# Testing Strategy & Pipeline Amendments

## Testing Philosophy

Every module MUST have unit tests before integration. Integration tests MUST pass before E2E. All test suites MUST pass before merging to main.

The test suite MUST cover **both** the channel path AND the existing `claude -p` fallback path. No existing test should break.

## CI/CD Pipeline Amendments

### GitHub Actions / CI Config

Add a new job or step for channel-specific testing:

```yaml
# In .github/workflows/test.yml (or equivalent)

test-channel:
  runs-on: ubuntu-latest
  env:
    CLAUDE_CHANNEL_ENABLED: true
    CLAUDE_CHANNEL_WS_PORT: 3199     # Non-default to avoid conflicts
    CLAUDE_CHANNEL_HOOK_PORT: 3198
  steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-node@v4
      with:
        node-version: 20
    - run: npm ci
    - run: npm run build
    - run: npm test
    - run: npm run test:e2e -- --grep "claude-channel"
```

## Regression Test Suite

Before final merge, run this full suite:

```bash
# 1. Clean build
npm run build

# 2. Lint + typecheck
npm run lint && npm run typecheck

# 3. All unit tests
npm test

# 4. Integration tests (channel path)
CLAUDE_CHANNEL_ENABLED=true npm test -- --testPathPattern="integration/claude"

# 5. Integration tests (claude -p path)
CLAUDE_CHANNEL_ENABLED=false npm test -- --testPathPattern="integration/claude"

# 6. E2E tests (channel path if Claude is available)
CLAUDE_CHANNEL_ENABLED=true npm run test:e2e -- --grep "claude"

# 8. Existing Pi + OpenCode E2E (must not regress)
```

## Known Test Dependencies on Claude CLI

Several existing tests check `claude` CLI behavior. These MUST continue working:

| Test | Depends On | Risk |
|---|---|---|
| `claude-process-pool.test.ts` | `claude` binary on PATH | Low — tests are mocked |
| `claude-service.test.ts` | `ClaudeProcessPool` mock | Low — all mocked |
| `claude-event-normalizer.test.ts` | Sample NDJSON fixtures | None — fixtures are static |
| `claude-history-replay.test.ts` | Sample JSONL entries | None — fixtures are static |
| E2E Claude tests | Real `claude` binary + auth | Medium — needs authenticated Claude |

For the channel E2E tests, a real `claude` binary with `--channels` support (v2.1.80+) is needed. If not available on CI, channel E2E tests should be skipped with a clear message.

---

## What NOT to Test

- **Don't test Claude Code internals.** Test only the Pi Web UI integration code.
- **Don't test MCP protocol internals.** The `@modelcontextprotocol/sdk` handles this.
- **Don't test Bun runtime.** Bun's behavior is assumed correct.
- **Don't end-to-end test the Claude → hook → HTTP receiver flow in unit tests.** This is integration test territory.
